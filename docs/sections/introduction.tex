\section{Introduction}
Choosing an appropriate time-stepper for a highly nonlinear partial differential equation (PDE) is a non-trivial task. Depending on the physical regime being modeled, systems can exhibit violently shifting spatial and temporal stiffness. This study utilizes the 1D \(p\)-Laplacian to evaluate solver robustness under these conditions:
\begin{equation} \label{eq:p-laplacian}
  \partial_t u = \nabla \cdot \left( (|\nabla u|^2 + \varepsilon^2)^{\frac{p-2}{2}} \nabla u \right)
\end{equation}
Varying the index \(p\) in Equation \eqref{eq:p-laplacian} transitions the model across linear, singular, and strongly degenerate regimes, making it an ideal stress-test for numerical integrators. 

However, evaluating these solvers requires looking beyond pure algorithmic complexity. Benchmark results are strictly contingent on the host architecture. Variations in L1/L2/L3 cache topologies, memory latency, CPU instruction sets, operating system schedulers, and underlying linear solver backends (such as KLU or LAPACK) can fundamentally alter relative execution speeds. This paper attempts to expose the mathematical limitations and hardware-level computational bottlenecks by varying system parameters and grid resolutions.
